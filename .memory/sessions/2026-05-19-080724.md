## Key Decisions

- **Scope-aware Phase 5 reads broadly, writes narrowly.** Chat scope selector is `shared | user | both` (default `both`); `propose_page` always writes to user's `authored/personal/`.
- **Two generation paths coexist**: Path A — implicit `propose_page` with inline preview + Save/Discard; Path B — post-hoc "Save as page" button that pre-fills the existing Editor via a new `initialDraft` prop. Each saves through existing CRUD.
- **Empty-source generation requires explicit opt-in.** Agent emits `{type: refuse, canForce: true}`; chat UI shows "Draft anyway (no sources)" button; opting in adds `forceUnsourcedGeneration: true` to the next request, which switches the system prompt into a forced-unsourced branch that drafts with an explicit "_No vault sources — drafted from general knowledge._" banner.
- **Citations are deterministic from `read_document` calls**, not text parsing. No-hallucination guarantee enforced at the protocol level.
- **Hybrid query strategy**: catalog (`_system/index.md`) embedded in the system prompt for global structure; `search_vault` available as a fallback tool for ambiguous queries.
- **Soft warning event** added in v2 to flag answers produced without any `read_document` calls (separate from the hard refusal that fires when search itself returned nothing).
- **Conversation history** sent on every `/api/chat` request (last 10 turns, intro/streaming excluded). Multi-turn context was missing in v1.
- **Bedrock model unchanged** — same `eu.amazon.nova-2-lite-v1:0` via the same `BEDROCK_MODEL` / `BEDROCK_REGION` env vars used by the curate Lambda.
