# Session — 2026-05-30 13:19:29

## Summary
Fixed a Bedrock Converse tool-result bug in the Ask-Wiki chat agent, then designed
and implemented a per-feature feature-flag system (env-controlled, all enabled by
default) gating both the UI and the API routes for every shipped Vaultmark feature.

## Key Decisions
- **Flag granularity:** fine-grained, per-feature (8 flags: agent, upload, curate,
  reindex, editor, search, star, publishing).
- **Control plane:** env vars only (`FEATURE_*`), read once at module load.
- **Default semantics:** ON unless var is `off`/`false`/`0`/`no`/`disabled` — absent ⇒ on,
  so all features ship enabled and current behavior/pixel-parity is unchanged.
- **Two-layer rule (non-negotiable):** every flag gates the UI entry point AND the
  matching route handler (404 via `flagGuard`). Hiding the button alone isn't control.
- Document read paths (`GET /api/docs`, tree, raw) are never gated — portal stays browsable.
- `publishing` is a forward-declared placeholder (Phase 8, no code/surface yet).

## Outcomes
- **Bug fix** in `web/lib/agent.ts`: `toolResultToJson` now wraps non-object results.
  Bedrock requires `toolResult.content[].json` to be a JSON object; `search_vault`
  returned a bare array → `messages.4.content.0.toolResult.content.0.json is invalid`.
  Arrays → `{ results: [...] }`, primitives/null → `{ value: ... }`.
- **New `web/lib/flags.ts`**: `FLAGS`, `isEnabled()`, `flagGuard(name)` (server-only,
  imports next/server; clients use type-only imports + prop-passing).
- **Route guards added**: chat, upload, reindex, search, curate (start/status/finalize/
  cancel), star (PATCH), docs (POST/PUT/DELETE; GET left open).
- **UI threading**: `FLAGS` → `app/page.tsx` + `app/[...id]/page.tsx` → `AppShell` →
  TopBar, Sidebar, HomeView, DocReader/DocToolbar, GeneratedDocReader, UploadModal.
  Gated buttons, tabs, ask-hero card, and ⌘K / ⌘⇧A keyboard shortcuts.
- **Docs**: feature-flag table added to `CLAUDE.md`; commented `FEATURE_*` entries
  added to `web/.env.local`.
- Typecheck + lint pass (both run `tsc --noEmit`).

## Architecture note
The real implementation lives in a **Next.js `web/`** app (App Router, route handlers,
`web/lib/*`), not the FastAPI `api/` described as the target in CLAUDE.md/steering docs.
The chat agent uses the Bedrock Converse-stream API with in-process tools
(search_vault, read_document, propose_page).
