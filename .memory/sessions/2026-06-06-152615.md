## Summary

Diagnosed why every curated source-card ended up flat under `generated/wiki/sources/`. Fixed the underlying multi-space routing bug inline, then planned and shipped a deterministic, rawKey-derived placement scheme that nests source pages under `sources/<placement>/<slug>.md`. Includes a non-destructive migration script for the existing 158 pages and a sidebar badge for last-curated visibility.

## Key Decisions

- **Multi-space routing fix.** `ingest.ts` now resolves output space from `card.suggestedSpaces` against a list of allowed generated spaces passed in by `index.ts`. `spaceFromRawKey` removed from the flow (pattern never matched the actual raw keys). Both shipped in the same change.
- **Placement is deterministic, not LLM-chosen.** Audit caught that LLM-chosen placement breaks citation stability, costs Bedrock tokens, drifts taxonomy, and requires a seed list. Replaced with `placementFromRawKey(rawKey)` — a pure function that mirrors `raw/`'s structure.
- **No `manifestVersion` bump, no force-reprocess.** Migration is pure S3 plumbing (CopyObject + manifest entry rewrite + DeleteObject). `processedAt` is preserved, no Bedrock cost, incremental processing semantics untouched.
- **Open synthesis categories deferred.** No rollups exist yet — opening the `concepts | features | projects` union now would be premature. Re-evaluate after synthesis runs against the foldered tree.
- **`folders.sources`/`folders.categories` in `structure.json` dropped.** Not needed under the deterministic design.
- **Spec rewritten and renamed.** Original `sources-foldering-and-categories.md` became `sources-foldering.md` after the audit shifted scope.
