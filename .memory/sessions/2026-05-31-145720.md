## Key Decisions

- **MOCK_S3 dispatch facade in `web/lib/s3.ts`** rather than refactoring all
  route handlers — keeps the test seam single-file and gated to one env var.
- **In-memory mock backed by `globalThis`** so state survives Next dev HMR
  and is shared across route handlers within one server process.
- **Production `next start` for tests, not `next dev`** — Turbopack HMR was
  racing React 19 hydration in headless Chromium and leaving the UI
  unhydrated. Production hydrates cleanly and is faster.
- **`export const dynamic = 'force-dynamic'` on `app/page.tsx` and
  `app/[...id]/page.tsx`** — without this, the home page was prerendered
  at build time with an empty tree, so the e2e fixture seed never
  surfaced. Defensible production change (tree depends on live S3).
- **Mock chat + curate at the Playwright network layer**, not server-side.
  These hit Bedrock/Lambda which would need their own mock implementations;
  page.route is one-tenth the code and equally e2e.
- **Two `webServer` entries, two projects**: main suite (all flags ON) +
  `flags-off` project on port +1 with every `FEATURE_*=off`. Lets the
  guard-404 behavior be tested without runtime env-toggling.
- **Renamed `__test__` → `test-seed` route folder** because Next App Router
  treats `_*`-prefixed dirs as private (silently un-routed).
